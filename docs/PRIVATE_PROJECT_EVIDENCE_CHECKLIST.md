# Private Project Evidence Checklist

Before publishing a case study:

- [ ] Replace names, emails, phone numbers and staff IDs with synthetic values.
- [ ] Remove bank accounts, payment references and transaction descriptions.
- [ ] Remove SharePoint tenant names, folder URLs, drive IDs and site IDs.
- [ ] Remove API keys, tokens, passwords, `.env` files and SMTP settings.
- [ ] Review screenshots for browser tabs, file paths and notification pop-ups.
- [ ] Use a simplified architecture diagram rather than production topology.
- [ ] Publish aggregate test results, not confidential fixtures.
- [ ] Confirm that code ownership permits public disclosure.
- [ ] Keep proprietary rules and client-specific mappings private.
- [ ] Offer a controlled walkthrough rather than unrestricted source access.
