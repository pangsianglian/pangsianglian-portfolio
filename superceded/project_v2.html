Executive Summary

During my internship at Tzu Chi Foundation Singapore, I built a production-oriented cashflow intelligence system that transforms fragmented financial files into a reliable, daily operational view. The solution ingests Excel-based inputs into SQL, runs a curated ETL pipeline, delivers Power BI dashboards for daily cash visibility, and automates bank statement auto-remark with a safe, auditable workflow.

On top of the data foundation, I implemented a low-balance risk detection module with ML-assisted logic and an email alerting system to notify Finance users early, enabling faster response and better cash planning.

Before vs After

Before

  • Cash visibility arrived late and required manual consolidation
  • Multiple spreadsheets with inconsistent formats and naming
  • Manual reconciliation was time-consuming and error-prone
  • Operational knowledge lived in people, not in a system
  • No consistent early-warning for low-balance situations

After

  • Daily operational visibility in Power BI dashboards
  • Standardized SQL staging + curated tables as source of truth
  • Automated auto-remark workflow for bank statement transactions
  • Designed for auditability, handover, and maintainable operations
  • Low-balance risk detection + email alerts to Finance users

Architecture at a Glance

Core Components

  • Input layer: Excel / statements
  • Data layer: SQL staging + curated schemas
  • Processing layer: ETL jobs + automation scripts
  • Analytics layer: Power BI datasets + dashboards
  • Intelligence layer: auto-remark + low-balance risk module
  • Notification layer: email alerts to Finance users

Design Principles

  • Reliability: repeatable runs and observable logs
  • Auditability: traceable decisions (rules/ML)
  • Human-in-the-loop: automation with safe review points
  • Maintainability: modular pipeline, clear handover paths
  • Scalability: extend tables, rules, and models as needs grow

Key Lessons

  • Finance problems become data problems — then become system design problems.
  • Automation only works when users trust the workflow and controls are built in.
  • ML value is realized when embedded into operations with review and auditability.
  • Handover-ready engineering matters as much as correctness.
